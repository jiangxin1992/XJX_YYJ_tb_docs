修改面料状态
=======

**url**: /service/fabric/updateStatus
**method**: post
**权限**: 设计师/销售代表


### 请求参数

|  名称  |  类型  | 是否必填 |               说明               |
|--------|--------|----------|----------------------------------|
| id     | number | Y        | 被修改的面料id, 多个时用逗号隔开 |
| status | number | Y        | 状态: 0 启用 ; 1 删除 ; 2 禁用   |

### 返回字段说明

1. 删除成功的返回值
```json
{
    "data":0
    "status":100
}
```

2. 部分删除成功的返回值
```json
{
    "data":1,
    "status":100
}
```
3. 删除失败的返回值
```json
{
    "message":"面料被款式引用，无法删除",
    "status":203
}
```

综上: 当`status=100`时需要刷新页面；当`data=1`时需要提示用户**部分面料被款式引用，无法删除**
